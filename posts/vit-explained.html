MY NOTES ON THE VISION TRANSFORMER, WRITTEN OUT PROPERLY

Nobody tells the model which patch is the wolf's eye. Self-attention figures that out — by letting every patch look at every other patch, all at once.

I went through "An Image Is Worth 16x16 Words" line by line, sat with the GeeksforGeeks breakdown, and traced back to what I'd already learned about the original Transformer — because ViT genuinely is just that architecture wearing a different hat. This is that — patches, positions, and everything in between — written the way I actually want to remember it.

01 · The Big Picture, Before Any Math

What ViT actually replaced

Before this paper, computer vision was CNN territory, full stop. Convolutions slide small filters across an image, building up understanding layer by layer — a pixel only ever sees its immediate neighborhood at first, and it takes many stacked layers before information from opposite corners of the image can influence each other at all.

"An Image Is Worth 16x16 Words: Transformers for Image Recognition at Scale" — Dosovitskiy et al., Google Research Brain Team, ICLR 2021 — asks a blunt question: does vision actually need any of that convolutional machinery, or can a plain, unmodified Transformer just look directly at raw image patches and do the job itself? Turns out, at large enough scale, yes — their best model hit 88.55% top-1 accuracy on ImageNet, 94.55% on CIFAR-100, and 77.63% across the 19-task VTAB suite, all while taking substantially less compute to pre-train than the best CNNs of the time.

The core move is almost embarrassingly simple: chop the image into fixed-size patches, treat each patch exactly like a word token, and feed the whole sequence into the same Transformer encoder architecture used for text. No convolutions anywhere except, arguably, the one linear projection that turns a patch into a vector.

CNN vs ViT, in one line A CNN assumes that nearby pixels matter more than distant ones, and bakes that assumption into every layer. A ViT assumes nothing about which patches matter to which — it has to learn every spatial relationship from data, using self-attention to let any patch talk to any other patch from layer one.
02 · Turning Pixels Into Tokens

Image patching — splitting and flattening

A Transformer can't read a 2D grid of pixels any more than it can read raw letters. Before any attention happens, the image has to become a 1D sequence — and that happens in two small steps, same shape as tokenization + embedding on the text side.

Take an image \(\mathbf{x} \in \mathbb{R}^{H \times W \times C}\). It gets divided into a grid of fixed-size, non-overlapping patches, each of size \(P \times P\). For a 224×224 image with patch size 16, that's a grid of \(224/16 = 14\) patches per side — 14 × 14 = 196 patches total, and that count becomes the effective sequence length \(N\) that the Transformer actually sees:

\[ N = \dfrac{HW}{P^2} \]

Each patch, of shape \(P \times P \times C\), is then flattened — spatial structure temporarily thrown away — into a single 1D vector of length \(P^2 \cdot C\):

\[ \mathbf{x}_p \in \mathbb{R}^{N \times (P^2 \cdot C)} \]
Why flatten at all? The Transformer encoder has no idea what "2D" means — every sub-layer just operates on a sequence of vectors. Flattening removes the spatial dimensions on purpose, so a 16×16×3 patch becomes one 768-number vector the model can treat uniformly, exactly like a word gets treated as one embedding vector regardless of how many letters it had.
03 · Giving Each Patch A Learned Identity

Patch embedding — the linear projection

A flattened patch is just raw pixel values — 768 numbers with no learned meaning yet. Exactly like a word needs an embedding lookup before a Transformer can use it, a patch needs to be projected into the model's working space.

Every flattened patch vector \(\mathbf{x}_p^i\) is multiplied by one trainable matrix \(E\), mapping it from \(P^2 \cdot C\) dimensions into the constant latent size \(D\) used by every layer in the model — \(D = 768\) for the Base config:

\[ \mathbf{z}_0 = [\mathbf{x}_{class};\; \mathbf{x}_p^1 E;\; \mathbf{x}_p^2 E;\; \cdots;\; \mathbf{x}_p^N E] + E_{pos} \]

where \(E \in \mathbb{R}^{(P^2 \cdot C) \times D}\). This is a single dense layer — nothing fancier than that. The output of this projection is what the paper calls the patch embeddings, and it's directly analogous to a word embedding table in NLP: a learned lookup that converts a raw unit into a meaningful vector.

A neat implementation shortcut In practice, this "split into patches, flatten, then linearly project" pipeline can be implemented as a single convolution layer with kernel size and stride both equal to the patch size. Each convolution slide then acts as exactly one non-overlapping patch extractor + projector — same math, one line of code.
04 · One Extra Token That Does All The Listening

The learnable [class] token

After projection you have 196 patch embeddings — but a classifier needs one single vector to make a decision from, not 196 of them. ViT's answer, borrowed straight from BERT, is to invent a 197th token that isn't a patch at all.

A learnable embedding is prepended to the sequence of patch embeddings, \(\mathbf{z}_0^0 = \mathbf{x}_{class}\). It carries no image content of its own at the start — it's just a vector of trainable parameters. But because self-attention lets every token attend to every other token, this class token gets to look at all 196 patches, layer after layer, and gradually accumulate a global summary of the whole image.

Why not just average-pool the patches instead, like a CNN would? The paper actually tried that — global average pooling over patch embeddings, followed by a linear classifier. It performed poorly at first, but purely because it needed a different learning rate, not because of the pooling itself. They kept the CLS token approach throughout the main paper to stay as close as possible to the original text Transformer, and because it's exactly what BERT already does successfully.

By the output of the final encoder layer, the class token's state \(\mathbf{z}_L^0\) is treated as the image representation \(\mathbf{y}\), and that's the only vector that gets passed to the classification head — the other 196 output vectors are simply discarded at inference time for classification.

05 · Giving The Model A Sense Of Where

Positional encoding

Here's the same real problem from the text Transformer, showing up again: self-attention looks at every patch simultaneously and has no built-in notion of "top-left" or "bottom-right." Without fixing this, the model would treat the image as an unordered bag of patches — shuffle the grid and it wouldn't even notice.

ViT's fix is simpler than the original paper's sinusoidal formula: standard learnable 1D position embeddings, one vector per position, added directly to each patch embedding (and to the class token too):

\[ E_{pos} \in \mathbb{R}^{(N+1) \times D} \]
Did fancier 2D-aware position embeddings help? Barely at all. The paper tried no positional info, 1D (default), 2D (separate row/column embeddings), and relative position embeddings — and found little to no difference between them, only a large gap versus using none at all. Their read: because the Transformer works on patch-level inputs (a 14×14 grid, not 224×224 pixels), the spatial dimensions are small enough that the model easily learns row/column structure from plain 1D position embeddings alone.
06 · The Part That Does All The Thinking

The Transformer encoder — Pre-LayerNorm architecture

Once every patch is a position-aware, D-dimensional vector, the actual "understanding" happens in a stack of \(L\) identical Transformer encoder blocks — the exact same block used on the text side, unmodified.

ViT uses Pre-LN: LayerNorm is applied before each sub-layer rather than after, and a residual connection wraps around each sub-layer:

\[ \mathbf{z}'_\ell = \text{MSA}(\text{LN}(\mathbf{z}_{\ell-1})) + \mathbf{z}_{\ell-1} \qquad \mathbf{z}_\ell = \text{MLP}(\text{LN}(\mathbf{z}'_\ell)) + \mathbf{z}'_\ell \]
Why Pre-LN instead of Post-LN? Normalizing before feeding into attention/MLP, rather than after adding the residual, keeps gradients well-behaved as the stack gets deep. This stabilizes gradient flow and helps avoid the exploding/vanishing gradient problem that deeper Transformer stacks are prone to — a small architectural choice that matters a lot once \(L\) gets into the dozens of layers.
\[ \text{LayerNorm}(x) = \dfrac{x - \mu}{\sigma} \odot \gamma + \beta \]

where \(\mu, \sigma\) are the mean and standard deviation computed across the feature dimension, and \(\gamma, \beta\) are learned scale and shift parameters.

07 · The Core Operation, Again

Self-attention over patches — Q, K, V

Inside "Multi-Head Attention" above is the exact same mechanism from text Transformers — just now the tokens being compared are image patches instead of words.

The input sequence is \(N\) patches plus 1 class token, each a \(D\)-dimensional vector, stacked into a matrix \(X\). Three learned weight matrices project \(X\) into Queries, Keys, and Values:

\[ Q = XW^Q, \qquad K = XW^K, \qquad V = XW^V \]
Q, K, V for patches — same search-engine analogy Query: "what am I, the ear patch, looking for elsewhere in the image?" Key: "what do I, the eye patch, have to offer, as a label?" Value: "what's my actual visual content?" The ear patch's Query gets compared against every other patch's Key to decide how much of that patch's Value to pull in — exactly the same as words, just applied to 16×16 pixel blocks.

Scores, scale, softmax, weight the Values — same five steps, same formula:

\[ \text{Attention}(Q,K,V) = \text{softmax}\!\left(\dfrac{QK^T}{\sqrt{d_k}}\right)V \]

\(QK^T\) computes a similarity score between every pair of tokens via dot product. \(d_k = D/h\) is the dimension per attention head, and dividing by \(\sqrt{d_k}\) keeps those scores from growing so large that softmax saturates into a region with vanishing gradients. Softmax turns the scaled scores into a probability distribution per token, and multiplying by \(V\) produces a weighted sum — each patch's new representation is a blend of every other patch's content, weighted by relevance.

This is the whole reason ViT can integrate information globally even in its very first layer, something a CNN's small local kernels simply cannot do without many stacked layers to grow the receptive field.

08 · Many Lenses On The Same Image

Multi-head self-attention (MSA)

One attention calculation gives every patch a single lens to compare itself against the rest of the image through. Multiple heads give it several lenses at once, each free to specialize.

Instead of one attention function over the full \(D\)-dimensional Q/K/V, the model runs \(h\) attention heads in parallel, each on a smaller \(D/h\)-dimensional slice, then concatenates and projects back:

\[ \text{head}_i = \text{Attention}(XW_i^Q, XW_i^K, XW_i^V) \] \[ \text{MSA}(X) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)\,W^O \]
Why bother with multiple heads for images? Different heads end up specializing in different types of relationships — one head might track edges, another color contrast, another whole-object shape or texture. In fact the paper's own attention-distance analysis found some heads attend to nearly the entire image even in the lowest layers, while others stay tightly localized near the query patch — the model uses both short-range and long-range attention simultaneously, and multi-head attention is what makes room for that split.
09 · Where Each Patch Processes What It Just Learned

The feed-forward network (FFN / MLP block)

After attention decides which patches are relevant to which, each token still needs a chance to actually transform that information — independently, with no cross-patch interaction this time.

Two dense layers with a GELU non-linearity in between, applied identically and independently to every token in the sequence:

\[ \text{FFN}(x) = W_2\,\text{GELU}(W_1 x + b_1) + b_2 \]

The inner dimension expands well beyond \(D\) — for ViT-Base, \(D_{MLP} = 3072\) versus \(D = 768\) — before projecting back down. GELU is used instead of ReLU here for a smoother, better-behaved non-linearity that tends to improve learning stability in deep Transformer stacks.

Attention vs FFN, once more Attention answers "which other patches should I pay attention to?" The FFN is where the token actually does something with that gathered context — a per-token, position-wise transformation with no communication between patches at this step. Same split in responsibilities as the text Transformer.
10 · Why Nothing Falls Apart At Depth

Residual connections and LayerNorm, together

Stack enough raw attention and MLP blocks without any safety net and training a deep network like this becomes unstable fast. Two small additions fix that, wrapped around every single sub-layer.

  • Residual (skip) connections bypass each transformation block, so the original signal is never fully overwritten — the model only has to learn an incremental adjustment to \(X\), not a full replacement, which is a much easier optimization problem.
  • LayerNorm rescales every token vector to zero mean and unit variance (with learned \(\gamma, \beta\)), keeping activations in a consistent range no matter how deep the stack gets — this is what makes training a 12-, 24-, or 32-layer ViT numerically stable instead of exploding or vanishing.

Together they're the reason a 32-layer ViT-Huge trains at all — remove either one and gradients either explode, vanish, or the model simply stops improving past a handful of layers.

11 · Turning A Vector Back Into A Label

The MLP classification head

After L encoder blocks, we're left holding one 768-dim vector — the final state of the class token, \(\mathbf{z}_L^0\). Somehow that has to become an actual class prediction.

\[ \mathbf{y} = \text{LN}(\mathbf{z}_L^0) \]

The classification head reads only this vector — not the 196 patch outputs — and passes it through a small feed-forward network: an MLP with one hidden layer (using tanh) during pre-training, collapsing to a single linear layer during fine-tuning. Softmax then converts the raw logits into class probabilities that sum to 1:

\[ P(\text{class}) = \text{softmax}(\text{Head}(\mathbf{y})) \]

Whichever class gets the highest probability is the prediction, and cross-entropy loss against the true label is what drives training end to end — patch embeddings, position embeddings, every attention weight, and the head itself, all learned jointly from the same signal.

12 · The Part That Actually Makes Or Breaks ViT

Training Vision Transformers — the data story

Everything above is architecture. None of it explains why ViT was considered risky in 2020, or why it works at all. That comes down to one property: inductive bias, or the near-total lack of it.

A CNN bakes locality and translation equivariance into every layer by construction — it assumes nearby pixels matter more, before it's even seen a single training image. ViT makes almost no such assumption; the only 2D-specific step is cutting the image into patches in the first place. Every spatial relationship between patches has to be learned purely from data.

The direct consequence: data hunger Trained on ImageNet alone (1.3M images) without heavy regularization, ViT scores a few points below comparably-sized ResNets. Trained on ImageNet-21k (14M images), the gap nearly closes. Only at JFT-300M scale (303M images) does ViT clearly pull ahead of ResNets on both accuracy and pre-training compute. The paper's own framing: large scale training trumps inductive bias — but only once you actually have that scale.

The practical training recipe follows the now-familiar large-model playbook: pre-train on a large dataset (often self-supervised or supervised at massive scale) to learn general visual features, then fine-tune on the smaller downstream dataset with far fewer labels — usually replacing the pre-trained head with a zero-initialized layer matching the new number of classes, and often fine-tuning at a higher image resolution than pre-training used.

13 · Side By Side

ViT vs CNN, the direct comparison

PropertyCNNViT
Attention scopeLocal, via convolutional kernelsGlobal, via self-attention — from layer 1
Inductive biasStrong — locality, translation equivarianceMinimal — mostly learned from data
Data requirementWorks well on small/mid datasetsNeeds large-scale pre-training to shine
Feature learningHierarchical, built up graduallyContext-rich, long-range from the start
Compute patternSequential depth to grow receptive fieldParallel attention across the full sequence
Not a strict "ViT wins" story The paper's own scaling study found hybrids — a ResNet feature extractor feeding into a ViT — slightly outperform pure ViT at small compute budgets, though that gap vanishes as models get larger. CNNs remain the better choice whenever you're data- or compute-constrained; ViT is the better choice once you have both scale and a large enough pre-training set.
14 · The Three Sizes From The Paper

Model variants — Base, Large, Huge

ViT configurations are borrowed directly from BERT's Base and Large sizes, with a new, bigger Huge added on top. Notation like ViT-L/16 means the "Large" config with 16×16 patches.

ModelLayersHidden size DMLP sizeHeadsParams
ViT-Base1276830721286M
ViT-Large241024409616307M

Smaller patch size means a longer sequence, and sequence length scales the compute cost as roughly its square — so ViT-L/16 is meaningfully more expensive to train than ViT-L/32, even though the parameter count barely changes. The flagship ViT-H/14, pre-trained on JFT-300M, is what produced the paper's headline 88.55% ImageNet number.

15 · The Honest Trade-offs

Advantages and limitations

Advantages

  • Global context from the very first layer, no need to stack depth to grow a receptive field.
  • Scales beautifully — performance keeps climbing with more data and bigger models, without visibly saturating in the paper's tested range.
  • Parallel by nature — the whole Transformer architecture computes across the sequence at once.
  • Unified architecture — the same block handles images, patches, or tokens with no vision-specific redesign.

Limitations

  • Data-hungry — needs genuinely large pre-training sets to beat CNNs, not just "big for ImageNet."
  • Quadratic attention cost — compute and memory grow with the square of the number of patches.
  • No local bias — doesn't naturally exploit local pixel patterns, hurting sample efficiency on small data.
  • Hyperparameter-sensitive — patch size, embedding dimension, and head count all need careful tuning.

If you forget everything else, remember this

  • Patchify the image into fixed, non-overlapping squares — this is ViT's only 2D-specific step.
  • Flatten + linearly project each patch into a D-dim vector, just like a word embedding.
  • Prepend a learnable [class] token — its final state becomes the whole image's representation.
  • Add learnable position embeddings so the model knows which patch sat where.
  • Pre-LN Transformer encoder, stacked L times: Norm → MSA → residual → Norm → MLP → residual.
  • Self-attention lets every patch attend to every other patch, globally, from layer one.
  • Multi-head attention runs that h times in parallel with different learned projections.
  • MLP head reads only the final class token and outputs class probabilities via softmax.
  • ViT wins over CNNs only once pre-training data is large enough — below that, inductive bias still matters.
